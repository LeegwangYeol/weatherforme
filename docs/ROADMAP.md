# WeatherForMe 발전 계획 — 통근 기반 로드맵

> 기준 시나리오: **경기 광주 ↔ 용인 포곡** 정기 통근 (2026-07 작성)

## 0. 왜 통근 중심인가

현재 앱은 "내 위치에 비 오면 알림"까지 완성됐다. 하지만 통근자의 실제 질문은 다르다:

- ❌ "지금 여기 비 와?" → ✅ **"오늘 퇴근길에 비 와? 아침에 우산 들고 나가야 해?"**
- 출근길 날씨는 출발 전에, **퇴근길 날씨는 아침에** 알아야 우산을 챙긴다
- 출발지·경유지·도착지의 날씨가 서로 다를 수 있다

### 실측 데이터: 광주↔포곡 통근 코리도

기상청 5km 격자 기준 (`latLngToGrid` 검증값):

| 지점 | 격자 (nx, ny) |
|---|---|
| 경기 광주시청 | (65, 124) |
| 광주 초월읍 | (65, 123) |
| 광주 도척면 | (66, 122) |
| **용인 포곡읍** | **(64, 121)** |

직선 보간시 경로는 **(65,124) → (65,123) → (64,123) → (64,122) → (64,121)** 총 5개 격자를 지난다.
→ 격자가 5칸이면 **출발지와 도착지 날씨가 실제로 갈릴 수 있는 거리**. 경로 기능이 유효하다.

## 1. 현재 보유 자산 (완료된 것)

| 자산 | 상태 | 통근 기능에서의 역할 |
|---|---|---|
| 다중 관심 지역 (`savedLocations`) | ✅ | 집/직장 등록의 기반 — 역할 필드만 추가하면 됨 |
| 격자별 알림 쿨다운 (`userId:gridKey`) | ✅ | 지역별 독립 알림 이미 지원 |
| 비구름 지도 (PTY 판정 통일) | ✅ | 경로 오버레이 캔버스 기반 |
| QStash 10분 크론 + 상태 점검 카드 | ✅ | 시간대 기반 브리핑의 신뢰 기반 |
| 초단기예보 지점/격자 API 승인 | ✅ | 6시간 내 정밀 감지 |
| **단기예보 격자 `nph-dfs_shrt_grd` 승인** | ✅ | **+6시간 너머 퇴근길 예보의 핵심** (이미 승인됨!) |
| 단기예보 지점 `getVilageFcst` | ✅ (동일 서비스) | 3일 예보 — 아침에 퇴근길 조회 |

## 2. 로드맵

### Phase 1 — 출퇴근 브리핑 🎯 (킬러 기능, 최우선)

**목표: 아침에 푸시 한 방으로 "오늘 우산 여부" 종결**

```
07:00 아침 브리핑 ☀️→☔
「출근길(광주, 8시) 맑음 · 퇴근길(포곡→광주, 18~20시) 비 예보!
  나갈 때 우산 챙기세요」
```

- [x] 관심 지역에 **역할 지정**: 🏠 집 / 🏢 직장 (savedLocations에 `role` 필드)
- [x] **통근 설정**: 출근 시간대(기본 07–09), 퇴근 시간대(기본 18–20), 통근 요일(기본 월–금)
- [x] **아침 브리핑 푸시** (기본 07:00, 조정 가능): 출근 시간대 집 격자 + 퇴근 시간대 직장→집 격자의 강수 예보를 한 문장으로 요약. 비 없으면 발송 생략 옵션("비 올 때만")
- [x] **퇴근 임박 리마인더**: 퇴근 시간 1시간 전, 퇴근길 비 예보 있을 때만
- [ ] 조용 시간(기본 22–07 무음) / 주말 제외 옵션

**기술 포인트**
- 퇴근 시간대는 아침 기준 +10시간 → 초단기예보(6h) 범위 밖 → **단기예보 사용**
  - 지점: `getVilageFcst` (발표 02/05/08/11/14/17/20/23시 — 아침 브리핑은 05시 발표분)
  - 또는 이미 승인된 격자 `nph-dfs_shrt_grd`
- `UserRecord.commute` 스키마: `{ homeLocId, workLocId, morning: [7,9], evening: [18,20], days: [1..5], briefingHour: 7, briefingAlways: false }`
- 크론이 KST 시각을 보고 브리핑 창(07:00±10분)에서 하루 1회 발송 — 중복 방지 키 `wfm:briefed:{userId}:{YYYYMMDD}`
- 브리핑과 임박 알림의 SW `tag` 분리 (서로 덮어쓰지 않게)

**작업량**: 서버 반나절 + 설정 UI 반나절

### Phase 2 — 통근 경로 날씨 (광주↔포곡 라인)

**목표: 점(두 지점)이 아니라 선(경로)으로 감시**

- [x] 집–직장 사이 **경로 격자 자동 보간** (위 실측처럼 직선 보간, 중복 제거 → ~5격자)
- [x] **경로 타임라인 UI**: 출발지→도착지 격자를 가로 스트립으로, 시간대별 강수 아이콘 (출근/퇴근 시간대 하이라이트)
- [x] **비구름 지도에 경로 오버레이**: 🏠·🏢 핀 2개 + 연결선, 경로 격자 테두리 표시
- [x] 알림 문구 업그레이드: 도착지가 맑아도 "경로 중간(초월 부근) 비구름 통과 예정 ☔"

**기술 포인트**
- 경로 격자는 클라이언트에서 보간 계산 후 저장 (서버는 격자 목록만 받음 — 기존 `byGrid` 로직 재사용)
- 비구름 지도 창(41×41)에 두 지점이 다 들어옴 (3칸 차이) — 오버레이만 추가하면 됨

**작업량**: 1일

### Phase 3 — 출발 타이밍 조언

**목표: "언제 나가면 비를 피하나"**

- [ ] 10분 갱신 초단기 격자(`nph-dfs_vsrt_grd`)로 **출발 시각 시뮬레이션**: 지금 출발 vs +30분 vs +1시간, 경로 격자 통과 시각별 강수 비교
- [ ] 이동 시간 설정 (기본 30분) — **도착지 판정은 도착 예정 시각 기준** (현재는 전부 "지금" 기준)
- [ ] 토끼 멘트 연동: "지금 나가면 딱 좋아요!" / "30분만 늦게 나가면 비 피해요 ☔"

**작업량**: 반나절 (Phase 2 위에)

### Phase 4 — 계절·데이터 확장 (옵션)

- [ ] **저녁 브리핑**: "내일 아침 출근길" 예고 (단기예보는 3일 커버)
- [ ] 실황 격자 `nph-dfs_odam_grd` 활용신청 → 비구름 지도에 "지금" 프레임 추가
- [ ] 겨울 모드: 눈·결빙 통근 알림 강화 (PTY 3/7 + 기온 0°C 이하 조합)
- [ ] 기상특보 API (호우·대설·강풍) — 예특보 카테고리에서 추가 신청
- [ ] 미세먼지 (에어코리아, 별도 API 키) — 통근 브리핑에 한 줄 추가

## 3. 우선순위와 이유

**Phase 1부터.** 이유:
1. 데이터·인프라가 전부 준비돼 있음 (단기예보 격자까지 이미 승인)
2. 통근자 체감 가치 최대 — "아침 푸시 한 방" 은 매일 쓰는 기능
3. Phase 2·3은 Phase 1의 집/직장·시간대 설정 위에 쌓임

```
Phase 1 (브리핑) ──→ Phase 2 (경로) ──→ Phase 3 (타이밍)
     └────────────── Phase 4 (계절 확장, 병렬 가능)
```

## 4. 리스크 / 유의점

| 리스크 | 대응 |
|---|---|
| 단기예보는 1시간 단위·3시간 발표 → 초단기보다 성김 | 브리핑은 단기예보로, 임박 알림은 기존 초단기 유지 (이원화) |
| 브리핑 시각 크론 정확도 (10분 간격) | 07:00±10분 창으로 판정 + 일별 중복 방지 키 |
| PTY vs RN1 판정 (2026-07-10 버그 교훈) | **모든 강수 판정은 PTY 기준 통일** — 신규 코드도 동일 |
| 설정 UI 복잡도 증가 | 기본값을 강하게 (광주↔포곡 케이스면 역할 지정만 하면 끝) |
